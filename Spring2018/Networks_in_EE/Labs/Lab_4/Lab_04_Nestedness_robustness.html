1. Nestedness

There are several methods for calculting the nestedness of a matrix. NODF is the most widely used metric and we will focus on it. NODF is evaluated separently per rows and columns, and then these scores are combined to get the NODF of the whole network. You are encouraged to look at the rationale behind it and how it compares to other metrics [1,2]. Nestedness has also been suggested for weighted networks [3,4], but we will work with binary networks.

Both vegan and bipartite have a function to calculate nestedness already implemented. The vegan package is more flexible. A basic example:

data("memmott1999")
memmott1999_binary <- 1*(memmott1999>0)
nodf <- nestednodf(memmott1999_binary)
nodf
## N columns  : 42.53719 
## N rows     : 45.97197 
## NODF       : 42.84196 
## Matrix fill: 0.1513924
plot(nodf)

Using oecosimu in vegan, calculting nestedness of a web and comparing it to a null model is super-easy! First we will compare to an equiprobabe shuffling algorithm:

2. Network robustness

The question of how networks with different stuctures respond to failure of (or attack on) nodes has been fundamental in the broader network science. For example, attack on hub vs random nodes will result in fundamentally different failure dynamics. These concepts are naturally adapted in ecology. In our next exercise we will try to build an algorithm for simulating species extinctions in food webs. Yet to give you a sense of how such analysis can look like, lets first take a look at existing implementation in R for bipartite networks. In the following implemetation we simulate an extinction process of the plants. There are 3 possible scenarios to extinction [5]:

  1. Least connected > most connected
  2. Most connected > leaset connected
  3. Random
data("memmott1999")
visweb(memmott1999)

# Follow secondary extinction of pollinators, while removing plants.
ex_random <- second.extinct(memmott1999, method = 'random', participant = 'lower', nrep=100, details=F) # Replicate simulations because it is random.
ex_random[1:3,]
##      no ext.lower ext.higher
## [1,]  1         1       0.57
## [2,]  2         1       1.14
## [3,]  3         1       1.01
ex_least <- second.extinct(memmott1999, method = 'abundance', participant = 'lower', details=F)
ex_least[1:3,]
##      no ext.lower ext.higher
## [1,]  1         1          0
## [2,]  2         1          0
## [3,]  3         1          0
ex_most <- second.extinct(memmott1999, method = 'degree', participant = 'lower', details=F)
ex_most[1:3,]
##      no ext.lower ext.higher
## [1,]  1         1         10
## [2,]  2         1          6
## [3,]  3         1          3

We can then plot an Attack Tolerance Curve [5]. Pardon for the ugly plots, these are built-in and very difficult to tweak directly. The function returns the exponent \(a\) of \(y \sim 1 - x^a\). So \(a\) is a measure of extinction vulnerability. The more gradual the secondary extinctions, the lower the absolute value of \(a\). Or, phrased differently, large absolute values of \(a\) indicate a very abrupt die-off, indicative of high initial redundancy in the network. (from ?slope.bipartite).

par(mfrow=c(2,2))
slope.bipartite(ex_random, cex.axis=0.1) #fit a curve to the extinction data
## exponent 
##  2.52404
slope.bipartite(ex_least) #fit a curve to the extinction data
## exponent 
## 15.21714
slope.bipartite(ex_most) #fit a curve to the extinction data
##  exponent 
## 0.5751362

The area under the curve can be used to quantify robustness, as suggested by Burgos [6].

# Robustness
robustness(ex_random)
## [1] 0.7100615
robustness(ex_least)
## [1] 0.9263332
robustness(ex_most)
## [1] 0.3605161
